  • Constructor: A special method used to initialize a newly created object when it is instantiated.

Unit 3 - Boolean Expressions and if Statements

Learning Objectives

  • Write and evaluate Boolean expressions using relational operators: ==, !=, <, >, <=, >=.
  • Use logical operators: (&&, ||, !) to combine Boolean expressions.
  • Apply De Morgan's Laws to simplify or rewrite Boolean expressions.
  • Implement conditional control structures using if, if-else, and nested if statements.
  • Trace and debug code that uses Boolean logic and conditional statements.
  • Understand short-circuit evaluation in logical expressions.
  • Write programs that make decisions based on user input or other conditions.

Curriculum Standards

  • The unit will take approximately 11 to 13 class periods to complete.
  • This part of the curriculum contributes to a 15% to 17.5% weight in the AP exam.

Guiding Questions

  • What is a Boolean expression, and how is it used in programming?
  • How do relational operators help us compare values in Java?
  • What are logical operators, and how do they combine Boolean expressions?
  • How do if, if-else, and else-if statements control the flow of a program?
  • How can we use nested if statements to handle more complex decision-making?
  • What are De Morgan's Laws, and how can they help us simplify Boolean expressions?
  • How do we compare objects (like Strings) in Java?

Key Concepts

  • Boolean Data Type: A primitive data type in Java that can hold one of two values: true or false.
  • Relational Operators: Used to compare two values and produce a Boolean result.
  • Logical Operators: Used to combine or modify Boolean expressions, logical expressions evaluate to true or false and are commonly used in conditional statements.
  • Short-Circuit Evaluation: In expressions using && and ||, Java evaluates from left to right and stops as soon as the result is determined:
    • For &&, if the first condition is false, the whole expression is false.
    • For ||, if the first condition is true, the whole expression is true.
  • De Morgan's Laws: Rules for rewriting logical expressions:
    • !(A && B), is equivalent to !A || !B.
    • !(A || B), is equivalent to !A && !B.
  • Comparing Objects: Use .equals(), to compare object values (e.g., Strings), not ==.

Relational Operators

  • ==: equal to
  • !=: not equal to
  • <: less than
  • >: greater than
  • <=: less than or equal to
  • >=: greater than or equal to

    • Logical Operators

  • &&: logical AND
  • ||: logical OR
  • !: logical NOT

    Unit 4 - Iteration

    Learning Objectives

    • Understand and apply while loops and for loops in Java.
    • Use loops to traverse and process data.
    • Write nested loops to solve complex problems.
    • Trace and debug iterative code.
    • Use loops in combination with conditionals to solve real-world problems.
    • Recognize when iteration is the appropriate control structure.

    Curriculum Standards

    • The unit will take approximately 14 to 16 class periods to complete.
    • This part of the curriculum contributes to a 17.5% to 22.5% weight in the AP exam.

    Guiding Questions

    • What is the purpose of a loop in a program?

    Key Concepts

    • Iteration / Looping: allow code to repeat without rewriting the same instructions multiple times.
    • While loops: are used when the number of repetitions is unknown and depend on a condition being true.
    • For loops: are typically used when the number of iterations is known or can be controlled with a counter.
    • Nested loops: are loops inside other loops and are useful for working with grids, patterns, or multi-dimensional data.
    • Accumulators and counters: are variables used inside loops to keep track of totals or counts.
    • Break: exits a loop early.
    • Continue : skips the current iteration and moves to the next one.

    Common Applications

    Counting and Summing Values

    Finding Maximum or Minimum

    • Iterate through a list or array to find the largest or smallest value.

    Unit 5 - Writing Classes

    Learning Objectives

    • Design and create classes with appropriate instance variables and methods.
    • Understand and implement constructors to initialize objects.
    • Use access modifiers (public, private) to control access to class members.
    • Distinguish between static and instance methods and variables.
    • Use the this keyword to reference the current object.
    • Implement method overloading to provide multiple methods with the same name but different parameters.
    • Apply encapsulation principles to create well-designed classes.

    Curriculum Standards

    • The unit will take approximately 12 to 14 class periods to complete.
    • This part of the curriculum contributes to a 5% to 7.5% weight in the AP exam.

    Guiding Questions

    • What is a class, and how does it differ from an object?
    • Why is it important to design classes with appropriate instance variables and methods?
    • What is the purpose of a constructor, and when is it called?
    • How do access modifiers control access to class members?
    • When should you use static versus instance methods and variables?
    • How does the this keyword help in class implementation?
    • What is method overloading, and when is it useful?
    • How does encapsulation improve class design?

    Key Concepts

    • Class Design: Creating well-structured classes that encapsulate related data and behavior.
    • Instance Variables: Variables that belong to an instance of a class and hold the object's state.
    • Instance Methods: Methods that operate on an instance of a class and can access instance variables.
    • Constructors: Special methods used to initialize objects when they are created.
    • Access Modifiers: Keywords that control the visibility and accessibility of class members:
      • public: Accessible from anywhere
      • private: Accessible only within the same class
    • Static Members: Members that belong to the class itself rather than to any specific instance.
    • this Keyword: A reference to the current object instance, used to distinguish between instance variables and parameters.
    • Method Overloading: Having multiple methods with the same name but different parameter lists.
    • Encapsulation: The practice of hiding internal details and providing controlled access through public methods.

    Access Modifiers

    • public: Members are accessible from any class.
    • private: Members are accessible only within the same class.
    • Default (no modifier): Members are accessible within the same package (package-private).

    Constructor Types

    • Default Constructor: No-argument constructor automatically provided if no constructors are defined.
    • Parameterized Constructor: Constructor that takes parameters to initialize object state.
    • Constructor Overloading: Multiple constructors with different parameter lists.

    Unit 6 - Array

    Learning Objectives

    • Understand the concept of arrays as data structures for storing multiple values of the same type.

    • Traverse arrays using loops to process all elements.

    Curriculum Standards

    Guiding Questions

    • What is an array, and why are arrays useful in programming?

    Key Concepts

    • Array: A data structure that stores a fixed-size sequential collection of elements of the same type.
    • Index: A numerical value used to access specific elements in an array, starting from 0.
    • Array Declaration: Creating a reference variable for an array using syntax like int[] numbers;.
    • Array Creation: Allocating memory for array elements using new keyword.
    • Array Initialization: Assigning initial values to array elements either during creation or separately.
    • Array Traversal: Processing all elements in an array using loops, typically for or enhanced for loops.
    • Array Algorithms: Common operations such as searching, sorting, finding maximum/minimum, and summing elements.
    • Array Bounds: Valid indices range from 0 to length - 1; accessing outside this range causes ArrayIndexOutOfBoundsException.

    Array Operations

    Declaration and Creation

    • String[] names = new String[5]; - Create array of 5 strings

    Accessing Elements

    • numbers[0] = 42; - Assign value to first element
    • int first = numbers[0]; - Get value of first element

    • Dynamic Resizing: ArrayLists automatically resize themselves when elements are added beyond their current capacity.
    • Enhanced for Loop: Simplified syntax for traversing ArrayLists: for (Type element : arrayList).

    Key Concepts

    • 2D Array: A data structure that stores elements in a grid or table format with rows and columns.
    • Row and Column Indices: Two-dimensional indexing where the first index represents the row and the second represents the column.
    • Array Declaration: Creating a reference variable for a 2D array using syntax like int[][] matrix;.
    • Array Creation: Allocating memory for 2D array elements using new keyword with dimensions.
    • Array Initialization: Assigning initial values to 2D array elements either during creation or separately.
    • Nested Loops: Using nested for loops to traverse rows and columns systematically.
    • 2D Array Algorithms: Common operations such as searching, finding maximum/minimum, summing rows/columns, and matrix operations.
    • Jagged Arrays: Arrays where each row can have a different length, creating a "jagged" appearance.

    2D Array Operations

    Declaration and Creation

    • int[][] matrix = new int[3][4]; - Create 3x4 matrix of integers
    • String[][] grid = new String[5][5]; - Create 5x5 grid of strings
    • double[][] values = {{1.5, 2.7, 3.9}, {4.2, 5.8, 6.1}}; - Initialize with values

    Accessing Elements

    • matrix[0][0] = 42; - Assign value to first element (row 0, column 0)
    • int first = matrix[0][0]; - Get value of first element
    • int rows = matrix.length; - Get number of rows
    • int cols = matrix[0].length; - Get number of columns

    Key Concepts

    • Inheritance: A mechanism that allows one class to acquire the properties and methods of another class, promoting code reuse and establishing hierarchical relationships.
    • extends Keyword: Used to declare that a class is a subclass of another class: class Subclass extends Superclass.
    • super Keyword: Used to refer to the immediate parent class object, access parent class members, and call parent class constructors.
    • Method Overriding: Providing a specific implementation of a method that is already defined in the parent class, using the same method signature.
    • Polymorphism: The ability of objects of different types to be treated as instances of the same type through a common interface, enabling dynamic method binding.
    • Abstract Classes: Classes that cannot be instantiated and may contain abstract methods (methods without implementation) that must be implemented by subclasses.
    • protected Access Modifier: Members that are accessible within the same package and by subclasses in different packages.
    • Method Resolution Order: The process by which Java determines which method implementation to use when methods are overridden in inheritance hierarchies.

    Inheritance Syntax and Keywords

    Class Declaration

    • class Subclass extends Superclass - Create a subclass that inherits from a superclass
    • super() - Call the parent class constructor
    • super.methodName() - Call a parent class method

    Access Modifiers

    • private - Accessible only within the same class
    • protected - Accessible within the same package and by subclasses
    • public - Accessible from anywhere

    Common Mistakes

    Inheritance Issues

    • Forgetting to call super() in subclass constructors when the parent class doesn't have a no-argument constructor.
    • Incorrectly using super keyword to access parent class members.
    • Not understanding that private members of the parent class are not inherited.

    Method Overriding Errors

    • Changing the method signature when overriding (not real overriding).
    • Forgetting the @Override annotation.
    • Trying to override static or private methods.

    Polymorphism Misunderstandings

    • Confusing compile-time type vs runtime type of objects.
    • Not understanding which methods can be called based on the reference type.

    Key Concepts

    • Recursion: A programming technique where a method calls itself to solve a problem by breaking it down into smaller, similar subproblems.
    • Base Case: A condition that stops the recursion, preventing infinite loops. It represents the simplest form of the problem that can be solved directly.
    • Recursive Case: The part of the method that calls itself with modified parameters, moving toward the base case.
    • Call Stack: The data structure that keeps track of method calls, including recursive calls, using a last-in-first-out (LIFO) approach.
    • Stack Overflow: An error that occurs when too many recursive calls consume all available stack memory, usually due to missing or incorrect base cases.
    • Direct vs Indirect Recursion: Direct recursion occurs when a method calls itself directly; indirect recursion occurs when method A calls method B, which calls method A.
    • Tail Recursion: A special form of recursion where the recursive call is the last operation in the method, potentially allowing for optimization.

    Recursive Method Structure

    Essential Components

    • Base Case: Condition that stops the recursion
    • Recursive Case: Call to the same method with modified parameters
    • Progress Toward Base Case: Each recursive call should move closer to the base case

    Example Structure

    public int factorial(int n) {
    // Base case
    if (n <= 1) {
        return 1;
    }
    // Recursive case
    return n * factorial(n - 1);
}

    Common Mistakes

    Base Case Issues

    • Missing or incorrect base cases leading to infinite recursion.
    • Base cases that don't properly handle edge cases (like negative numbers or zero).

    Recursive Case Problems

    • Recursive calls that don't move toward the base case.
    • Incorrect parameter modifications in recursive calls.

    Understanding and Tracing

    • Difficulty visualizing the call stack and execution flow.
    • Confusing the order of operations in recursive methods.
    • Not understanding how return values propagate back up the call stack.
