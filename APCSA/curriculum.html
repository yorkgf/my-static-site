Unit 1 - Primitive Types

Learning Objectives

  • Understand the reasons for programming and why Java is chosen as the teaching language.
  • Learn about variables and data types.
  • Understand the role of expressions and assignment statements.
  • Master the use of compound assignment operators.
  • Understand the ranges and type casting of variables.

Curriculum Standards

  • The unit will take approximately 8 to 10 class periods to complete.
  • This part of the curriculum contributes to a 2.5% to 5% weight in the AP exam.

Guiding Questions

  • Why is Java chosen as the primary language for teaching in AP Computer Science A?
  • What are variables, and how do they store data?
  • What are the different data types in Java, and when should each be used?

Key Concepts

  • Reasons for choosing Java: platform independence, robustness, large community support.
  • Data types in Java: primitive types (int, double, boolean, etc.), reference types (objects).
  • Importance of data types: ensuring correct data handling and preventing errors.
  • Common operators: arithmetic (+, -, *, /), relational (<, >, ==, !=), logical (&&, ||, !) .
  • Type casting: converting one data type to another.

Key Vocabulary

  • Variable: A named storage location that holds a value.
  • Data Type: A classification that defines the type of data a variable can hold.
  • Primitive Data Types: Basic data types in Java, such as int, double, boolean, etc.
  • Reference Data Types: Data types that refer to objects, such as String or custom classes.

Unit 2 - Using Objects

Learning Objectives

  • Understand the concept of objects as instances of classes.
  • Learn how to create and store objects (instantiation).
  • Master the process of calling void methods.
  • Practice calling void methods with parameters.
  • Learn to call non-void methods.
  • Explore string objects, including concatenation, literals, and more string methods.
  • Study wrapper classes for Integer and Double.
  • Utilize the Math class for various mathematical operations.

Curriculum Standards

  • The unit will take approximately 13 to 15 class periods to complete.
  • This part of the curriculum contributes to a 5% to 7.5% weight in the AP exam.

Guiding Questions

  • What is the difference between a class and an object?
  • Why is it important to store objects in variables?
  • What is a void method, and why are they useful?
  • What is the purpose of passing parameters to a void method?
  • What distinguishes a non-void method from a void method?
  • Explain the concept of string concatenation and provide an example.
  • What are wrapper classes, and why are they used?
  • What methods are commonly found in the Math class in Java?

Key Concepts

  • Class: A blueprint or template that defines the properties (fields) and behaviors (methods) of objects.
  • Object: An instance of a class, which is a specific entity that has its own state and behavior.
  • Instantiation: The process of creating an object from a class using a constructor.
  • Variables: Stores references to objects, allowing you to manipulate them.
  • Parameters: Values passed to a method during its execution.
  • Constructor: A special method used to initialize a newly created object when it is instantiated.
  • Return Value: The result produced by the method, which can be used in further computations.

Unit 3 - Boolean Expressions and if Statements

Learning Objectives

  • Write and evaluate Boolean expressions using relational operators: ==, !=, <, >, <=, >=.
  • Use logical operators: (&&, ||, !) to combine Boolean expressions.
  • Apply De Morgan's Laws to simplify or rewrite Boolean expressions.
  • Implement conditional control structures using if, if-else, and nested if statements.
  • Trace and debug code that uses Boolean logic and conditional statements.
  • Understand short-circuit evaluation in logical expressions.
  • Write programs that make decisions based on user input or other conditions.

Curriculum Standards

  • The unit will take approximately 11 to 13 class periods to complete.
  • This part of the curriculum contributes to a 15% to 17.5% weight in the AP exam.

Guiding Questions

  • What is a Boolean expression, and how is it used in programming?
  • How do relational operators help us compare values in Java?
  • What are logical operators, and how do they combine Boolean expressions?
  • How do if, if-else, and else-if statements control the flow of a program?
  • How can we use nested if statements to handle more complex decision-making?
  • What are De Morgan's Laws, and how can they help us simplify Boolean expressions?
  • How do we compare objects (like Strings) in Java?

Key Concepts

  • Boolean Data Type: A primitive data type in Java that can hold one of two values: true or false.
  • Relational Operators: Used to compare two values and produce a Boolean result.
  • Logical Operators: Used to combine or modify Boolean expressions, logical expressions evaluate to true or false and are commonly used in conditional statements.
  • Short-Circuit Evaluation: In expressions using && and ||, Java evaluates from left to right and stops as soon as the result is determined:
    • For &&, if the first condition is false, the whole expression is false.
    • For ||, if the first condition is true, the whole expression is true.
  • De Morgan's Laws: Rules for rewriting logical expressions:
    • !(A && B), is equivalent to !A || !B.
    • !(A || B), is equivalent to !A && !B.
  • Comparing Objects: Use .equals(), to compare object values (e.g., Strings), not ==.

Relational Operators

  • ==: equal to
  • !=: not equal to
  • <: less than
  • >: greater than
  • <=: less than or equal to
  • >=: greater than or equal to

    • Logical Operators

  • &&: logical AND
  • ||: logical OR
  • !: logical NOT

    Unit 4 - Iteration

    Learning Objectives

    • Understand and apply while loops and for loops in Java.
    • Use loops to traverse and process data.
    • Write nested loops to solve complex problems.
    • Trace and debug iterative code.
    • Use loops in combination with conditionals to solve real-world problems.
    • Recognize when iteration is the appropriate control structure.

    Curriculum Standards

    • The unit will take approximately 14 to 16 class periods to complete.
    • This part of the curriculum contributes to a 17.5% to 22.5% weight in the AP exam.

    Guiding Questions

    • What is the purpose of a loop in a program?
    • How do you determine when to use a while loop versus a for loop?
    • How does the loop control variable affect the behavior of a loop?
    • What happens if the loop condition is never false?
    • How can we use loops to process each character in a string?
    • How do nested loops work, and how do they affect program efficiency?

    Key Concepts

    • Iteration / Looping: allow code to repeat without rewriting the same instructions multiple times.
    • While loops: are used when the number of repetitions is unknown and depend on a condition being true.
    • For loops: are typically used when the number of iterations is known or can be controlled with a counter.
    • Nested loops: are loops inside other loops and are useful for working with grids, patterns, or multi-dimensional data.
    • Accumulators and counters: are variables used inside loops to keep track of totals or counts.
    • Break: exits a loop early.
    • Continue : skips the current iteration and moves to the next one.

    Common Applications

    Counting and Summing Values

    • Use a loop to calculate the sum or average of a series of numbers.

    Finding Maximum or Minimum

    • Iterate through a list or array to find the largest or smallest value.

    String Traversal

    • Use a loop to examine or manipulate each character in a string.

    Common Mistakes

    Creating Infinite Loops

    • Forgetting to update the loop control variable inside a while loop.
    • Using a condition that never becomes false.

    Off-by-One Errors (OBOE)

    • Using incorrect loop bounds (e.g., starting at 1 instead of 0 or using <= instead of <).

    Using == to Compare Strings

    • Using == instead of .equals() when comparing String values inside loops.
