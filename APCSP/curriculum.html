Unit 1 - Digital Information

Learning Objectives

  • Understand how computers store and represent different types of digital information.
  • Explore binary representation of numbers, text, images, and sound.
  • Learn about data compression techniques and their applications.
  • Analyze the impacts of digitizing information on society and culture.
  • Debate the ethical implications of digital information storage and sharing.

Curriculum Standards

  • The unit will take approximately 8 to 10 class periods to complete.
  • This part of the curriculum contributes to a 10% to 15% weight in the AP exam.

Guiding Questions

  • How do computers store complex information like numbers, text, images, and sound?
  • What are the advantages and limitations of digital representation?
  • How do data compression techniques work, and when are they useful?
  • What are the social and cultural impacts of digitizing information?
  • What ethical considerations arise from digital information storage and sharing?

Key Concepts

  • Binary Representation: How all digital information is stored using combinations of 0s and 1s.
  • Data Types: Different categories of digital information including numerical, textual, visual, and audio data.
  • Data Compression: Techniques for reducing the size of digital files to save storage space and bandwidth.
  • Digital Transformation: The process of converting analog information into digital format and its societal impacts.
  • Information Ethics: Moral principles governing the collection, storage, and sharing of digital information.

Key Vocabulary

  • Binary: A base-2 number system using only 0s and 1s to represent all digital information.
  • Bit: The smallest unit of digital information, representing a single binary digit (0 or 1).
  • Byte: A group of 8 bits, commonly used to represent characters or small numbers.
  • Compression: The process of reducing the size of digital files to save storage space or transmission time.
  • Metadata: Data that provides information about other data, such as file creation date or author.

Core Skills

  • Data Representation: 4
  • Computing Innovations: 5
  • Responsible Computing: 6

  • Quizzes

  • Data Representation Projects
  • Class Discussions

Summative Assessments

  • Unit Test
  • Digital Information Analysis Project

Applications

  • File Storage Systems: How computers store documents, images, and videos using binary representation in cloud storage services like Google Drive and Dropbox.
  • Digital Media: JPEG compression in digital cameras, MP3 encoding in music streaming services, and video compression in YouTube and Netflix.
  • Data Transmission: How information is encoded and transmitted across networks in applications like email, messaging apps, and online banking.
  • QR Codes: Real-world application of binary data representation in retail, payments, and inventory management systems.

Unit 2 - The Internet

Learning Objectives

  • Understand the fundamental architecture and protocols of the Internet.
  • Learn about IP addresses, DNS, and packet switching.
  • Explore how data travels across networks and the Internet.
  • Analyze the political, cultural, and economic impacts of the Internet.
  • Discuss Internet governance and digital rights issues.

Curriculum Standards

  • The unit will take approximately 10 to 12 class periods to complete.
  • This part of the curriculum contributes to a 15% to 20% weight in the AP exam.

Guiding Questions

  • How does the Internet work, and what technologies make it possible?
  • What are IP addresses, DNS, and how do they help us navigate the web?
  • How does data travel across the Internet, and what protocols are involved?
  • What are the political, cultural, and economic impacts of the Internet?
  • How should the Internet be governed, and what digital rights should be protected?

Key Concepts

  • Internet Architecture: The underlying structure and protocols that enable global network communication.
  • Network Protocols: Standardized rules and formats for data transmission across networks.
  • Packet Switching: The method of breaking data into packets for efficient transmission across networks.
  • Digital Society: How the Internet has transformed politics, culture, and economics globally.
  • Internet Governance: The development and application of shared principles, norms, and standards for the Internet.

Key Vocabulary

  • IP Address: A unique numerical label assigned to each device connected to a computer network.
  • DNS: Domain Name System, which translates human-readable domain names to IP addresses.
  • Packet: A unit of data transmitted over a network, containing both header and payload information.
  • Protocol: A set of rules and conventions for communication between network devices.
  • Bandwidth: The maximum rate of data transfer across a network connection.

Unit 3 - Intro to App Design

Learning Objectives

  • Design your first app using visual programming tools.
  • Learn fundamental programming concepts including events and user interfaces.
  • Understand collaborative software development processes.
  • Apply user-centered design principles in app development.
  • Practice iterative design and testing methodologies.

Curriculum Standards

  • The unit will take approximately 10 to 12 class periods to complete.
  • This part of the curriculum contributes to a 15% to 20% weight in the AP exam.

Guiding Questions

  • How do you design and create a basic mobile application?
  • What are the fundamental programming concepts used in app development?
  • How do collaborative software development processes work?
  • What principles guide effective user interface design?
  • How do you iteratively improve an app through testing and feedback?

Key Concepts

  • Visual Programming: Creating applications using graphical interfaces rather than traditional text-based coding.
  • Event-Driven Programming: A programming paradigm where the flow of the program is determined by events.
  • User Interface Design: The process of designing interfaces that are easy to use and provide a good user experience.
  • Collaborative Development: Working together in teams to design, build, and test software applications.
  • Iterative Design: A cyclical process of prototyping, testing, analyzing, and refining a product or process.

Key Vocabulary

  • Event: An action or occurrence detected by a program, such as a button click or screen tap.
  • User Interface (UI): The space where interactions between humans and machines occur.
  • Prototype: An early sample, model, or release of a product built to test a concept or process.
  • Collaboration: Working together to achieve a common goal, often involving diverse perspectives and skills.
  • Debugging: The process of identifying, isolating, and fixing errors or bugs in programs.

  • What are functions, and how do they help organize code?

Key Concepts

  • Variables: Named storage locations that hold values which can be changed during program execution.

  • API Integration: Connecting applications to external web services to access and use their data and functionality.
  • Code Organization: Structuring programs in a logical, readable, and maintainable way.

Key Vocabulary

  • Variable: A named storage location that holds a value which can be changed during program execution.
  • Conditional: A programming construct that executes different code paths based on whether a specified condition is true or false.
  • Function: A reusable block of code that performs a specific task and can be called multiple times with different inputs.
  • API: Application Programming Interface, a set of rules and protocols for building and interacting with software applications.
  • Scope: The region of a program where a variable or function is accessible.

Key Vocabulary

  • Traversal: The process of accessing each element in a data structure exactly once.

Unit 6 - Algorithms

Learning Objectives

  • Design and implement algorithms to solve computational problems.
  • Analyze algorithm efficiency and complexity using basic metrics.
  • Compare different algorithms for the same problem to determine effectiveness.
  • Understand common algorithmic patterns and problem-solving strategies.
  • Evaluate the trade-offs between different algorithmic approaches.

Curriculum Standards

  • The unit will take approximately 10 to 12 class periods to complete.
  • This part of the curriculum contributes to a 10% to 15% weight in the AP exam.

Guiding Questions

  • What makes an algorithm effective, and how do we design good algorithms?
  • How do we analyze and compare the efficiency of different algorithms?
  • What are common algorithmic patterns used in problem-solving?
  • How do we evaluate the trade-offs between different algorithmic approaches?
  • What role do algorithms play in modern computing applications?

Key Concepts

  • Algorithm Design: The process of creating step-by-step procedures to solve computational problems.
  • Algorithm Analysis: Techniques for evaluating the efficiency and correctness of algorithms.
  • Complexity Metrics: Methods for measuring how algorithm performance scales with input size.
  • Problem-Solving Strategies: Systematic approaches to breaking down and solving computational problems.
  • Algorithm Comparison: Techniques for evaluating and comparing different solutions to the same problem.

Key Vocabulary

  • Algorithm: A finite sequence of well-defined instructions for solving a problem or completing a task.
  • Efficiency: How well an algorithm uses resources such as time and memory to solve a problem.
  • Complexity: A measure of how the resource requirements of an algorithm grow as the input size increases.
  • Pseudocode: A simplified, informal language used to develop algorithms without specific syntax rules.
  • Trade-off: A situation where gaining one benefit involves losing another.

Unit 7 - Parameters, Return, and Libraries

Learning Objectives

  • Design functions with parameters to make them more flexible and reusable.
  • Implement return values to enable functions to produce output.
  • Use and create code libraries to share functionality across projects.
  • Practice writing clean, modular code that follows best practices.
  • Collaborate effectively by sharing and documenting code components.

Curriculum Standards

  • The unit will take approximately 8 to 10 class periods to complete.
  • This part of the curriculum contributes to a 5% to 10% weight in the AP exam.

Guiding Questions

  • How do parameters make functions more flexible and powerful?
  • What role do return values play in function design and program flow?
  • How do code libraries help developers work more efficiently?
  • What practices help create clean, maintainable, and reusable code?
  • How can developers effectively collaborate by sharing code components?

Key Concepts

  • Function Parameters: Input values that functions accept to customize their behavior.
  • Return Values: Output values that functions produce and send back to the calling code.
  • Code Libraries: Collections of pre-written code that can be reused across different projects.
  • Code Reusability: The practice of writing code that can be used in multiple contexts without modification.
  • Code Documentation: The process of writing clear explanations and instructions for code usage.

Key Vocabulary

  • Parameter: A named variable that receives a value passed to a function when it is called.
  • Return Value: The output or result that a function sends back to the code that called it.
  • Library: A collection of pre-written code modules that can be reused in different programs.
  • Modularity: The degree to which a system's components may be separated and recombined.
  • Documentation: Written explanations and instructions that help users understand and use code.
